# 知识点

## 数据总线与地址总线

主存最大容量由 地址总线位数 决定（还取决于“按什么单位编址”）；数据总线位数不决定容量，只影响一次传输的数据量/字长与带宽。

## 主存芯片的拓展

字拓展：利用译码器（串联）

位拓展：将每一位连接到不同的地址总线上（并联）

## 32位机

当我们说一台计算机是 **“32位机”** 时，这通常指的是其 **CPU（中央处理器）** 的核心架构。

具体来说，这意味着以下几个关键的硬件部件必须是 **32位** 宽度的：

### 1. 通用寄存器 (General Purpose Registers) —— **最核心的标志**

这是判定机器字长的 **金标准** 。

* CPU 内部用来暂存数据的小仓库（如 x86 架构下的 EAX, EBX 等）。
* **32位机**意味着这些寄存器每一个都能存储 **32位（4字节）** 的二进制数据。
* 如果寄存器只有 16 位，那就是 16 位机（如 8086）；如果是 64 位，就是 64 位机。

### 2. 运算器 (ALU - Arithmetic Logic Unit)

* ALU 是 CPU 负责计算（加减乘除、逻辑运算）的大脑。
* **32位机**意味着 ALU 可以**一次性**处理两个 **32位** 的数字进行运算。
* 不需要像 8 位机处理 32 位数据那样，拆分成 4 次 8 位运算来做。

### 3. 数据总线 (Data Bus) —— **通常情况**

* 这是 CPU 与内存之间传输数据的“高速公路”。
* 在标准的 32 位架构中，外部数据总线通常也是 **32位** 宽的。
* 这意味着 CPU **一口（一个时钟周期）** 能从内存里搬运 32 位的数据。
  * *注：历史上曾有“准32位机”（如 386SX），内部是 32 位，外部数据总线是 16 位，这种虽然也是 32 位机，但性能打了折扣。*

### 4. 寻址能力 (指针 / Program Counter)

这虽然不是一个物理部件的名字，但它对应了 **地址总线** 和 **指令指针寄存器 (PC)** 的宽度。

* **指针长度：** 程序中的内存地址（指针）是 32 位的。
* **寻址范围：** CPU 能识别的内存地址范围是 **$0 \sim 2^{32}-1$**。
* 这就是为什么 32 位 Windows 系统最大只能识别 **4GB** 内存的原因（**$2^{32} = 4,294,967,296$** 字节 = 4GB）。

  ### 存储器的模块化组织方式
* **顺序方式组织 (High-Order Interleaving)：**

  * **原理：** 连续的地址分布在同一个存储模块中（例如：第1、2、3、4个字都在模块0中，直到模块0装满才去模块1）。
  * **特点：** 由于连续读出的 4 个字都在同一个模块内，而一个模块在执行读写操作时是“忙碌”的，必须等待一个完整的存储周期 (**$T$**) 结束后，才能开始下一次读写。
  * **无法并行：** 因此，读取 **$n$** 个字需要 **$n$** 个完整的存储周期。
* **交叉方式组织 (Low-Order Interleaving)：**

  * **原理：** 连续的地址分布在不同的模块中（例如：字1在模块0，字2在模块1...）。
  * **特点：** 可以实现流水线式的并行读取。
  * **耗时公式：** **$T + (n-1)\tau$**。
